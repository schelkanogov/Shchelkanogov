## 🔄 User Journey (Полный путь пользователя)

### Этап 1: Точка входа
Пользователь приходит на **vulndetector.ru** (Landing) и заказывает бесплатный отчёт:
- Вводит домен + имя + email → лид сохраняется в `/api/lead`
- HTML-отчёт приходит на почту **It.immunirty@yandex.ru** (и пользователю)

### Этап 2: КиберКвиз (quiz.html)
Пока отчёт готовится (~20 мин), пользователю предлагается пройти квиз:
- **10 вопросов** по ключевым областям ИБ
- Категории: Инфраструктура, Аутентификация, Периметр, Бэкапы, Compliance, Команда, Мониторинг, Обучение, Патчи, IRP
- Каждый ответ = **0-3 балла** + тег для профилирования
- Результат: оценка 0-100%, слабые зоны, рекомендации
- Данные сохраняются в `localStorage` под ключом `vulndetector_quiz`

### Этап 3: Boardroom Simulator (index.html)
Пользователь попадает в виртуальную переговорку:

1. **Загрузка отчёта** — HTML-отчёт VulnDetector парсится клиентским JS:
   - Извлекается: IT-Immunity score, CVE, домен, кол-во уязвимостей
   - Данные обогащают диалоги всех агентов

2. **Выбор сценария** (= диагностическая карта):
   - 4 предустановленных: ФСТЭК, Ransomware, Бюджет, NGFW
   - Кастомный ввод
   - Если квиз пройден — баннер с результатами

3. **Симуляция** — 4 AI-агента обсуждают ситуацию:

   | Агент | Роль | Цвет | Эффект |
   |-------|------|------|--------|
   | 👮 Товарищ Майор | Регулятор, ФСТЭК | 🔵 Синий | Grayscale 15% |
   | 🥷 APT-Хакер | Злоумышленник | 🔴 Красный | Shake + Flash |
   | 👔 CFO | Бизнес | 🟡 Жёлтый | Grayscale 20% |
   | 🟢 Агент Сбера | Решение | 🟢 Зелёный | Full Color! |

4. **Механика "Красная нить"**:
   - Вокруг каждого агента орбитируют thought-tags
   - Клик по тегу = соединение красной нитью
   - 2+ соединения → Prediction Cloud (центральный прогноз)
   - Если один из тегов Сбера → "Риск нейтрализован"

### Этап 4: Paywall + Lead Capture
- **Без авторизации**: скачать резюме-отчёт (хроника + KPI, без КП)
- **С авторизацией** (имя + email обязательно):
  - Форма: Имя*, Email*, Телефон, Компания
  - При отправке:
    1. Лид сохраняется в `localStorage` + `leads.json`
    2. Лид отправляется на Kanban-webhook (POST JSON)
    3. Email-уведомление на It.immunirty@yandex.ru (SMTP)
  - Разблокируется: Экспертное КП (HTML) + полный отчёт + бонус-пакет

## ⚙️ Unified Backend (server.py)

### Запуск
```bash
# 1. Создать .env из шаблона
cp ../.env.example ../.env
# Заполнить ключи API

# 2. Установить зависимости
pip install fastapi uvicorn httpx aiosmtplib pydantic[email]

# 3. Запустить
cd CyberSec_Simulator
python server.py
# → http://localhost:8080/          Симулятор
# → http://localhost:8080/quiz       КиберКвиз
# → http://localhost:8080/landing/   Landing page
# → http://localhost:8080/panel/     UI Panel
```

### Multi-LLM каскад
```
GigaChat → Perplexity → Claude → Ollama → Static Fallback
```
При ошибке любого LLM автоматически переключается на следующий.

### API Endpoints
| Endpoint | Метод | Описание |
|----------|-------|----------|
| `/api/chat` | POST | AI-диалог с персоной (с RAG) |
| `/api/lead` | POST | Сохранение лида + Kanban + SMTP |
| `/api/stats` | GET | Статистика (vuln count, immunity) |

### RAG
- Источник: `Y:\!!! RaG!!!\03_Enriched_RAG\*.md`
- Keyword-scoring по запросу пользователя
- Контекст инъецируется в system prompt

## 🎨 Визуальная механика

### SVG-город (sceneWrapper)
- 7 зданий: Датацентр, Офис, Sber SOC, ФСТЭК, Хакерский подвал, Заказчик, Модерн
- Анимации: мигающие окна, движущиеся машины, дрон, Wi-Fi сигналы
- CSS Variable `--saturation`: 0 (ЧБ) → 1 (Full Color)
- Переход ЧБ→Цвет при действиях Агента Сбера
- Фоторелистичные фоны: bg_regulator/hacker/cfo/sber.png с fade-transition

### Частицы (particleCanvas)
- 60 частиц + сетка связей
- Цвет меняется по агенту: синий/красный/жёлтый/зелёный

### Web Audio API (SFX)
- `click` — 600Hz sine
- `danger` — 180Hz sawtooth  
- `win` — 880Hz sine
- `type` — ~1200-1600Hz sine (каждый 5-й символ)

## 🔐 Важные принципы

1. **КП только после аутентификации** — требование регулятора, воронка продаж
2. **Сценарий = диагностическая карта** — первичный опросник для клиентов, которые "многое не знают и не умеют заполнять"
3. **Все данные в лид** — контакты + квиз + отчёт + KPI + хроника → Kanban → прогрев
4. **Данные локальны** — вся обработка на клиенте, суверенность данных
5. **LLM-каскад** — система всегда отвечает, даже если все API недоступны (static fallback)
